Prototypal versus Classical Inheritance

Ali Groening

What is Inheritance?

  • dictionary.com: "something that is or may be inherited"
  • One obejct can access the properties and methods of another object.

Prototye

  • Prototype is a property of an object
  • The prototype property can hold properties and methods
  • Protoypes can be "linked" to other objects, specifically instances of the object, which creates "differential" inheritance
  • Objects can only "link"/inherit directly from other objects

Example of Prototypal Pattern


							const Person = {
								introduce: function () {
									console.log( `${ this.firstName } ${ this.lastName } is in da house!` );
								}
							};
							const ali = Object.create( Person, {
								firstName: { value: 'Ali' },
								lastName: { value: 'G' }
							} );
							ali.introduce(); //Ali G is in da house!

Function Constructor/Psuedo Classical Pattern


							function Person( firstName, lastName ) {
								this.firstName = firstName;
								this.lastName = lastName;
							};
							Person.prototype.introduce = function () {
								console.log( `${ this.firstName } ${ this.lastName } is in da house!` );
							};
							const ali = new Person( 'Ali', 'G' );
							ali.introduce(); //Ali G is in da house!

Classical

  • classes can only extend from other classes
  • classes create hierarchical class taxonomies
  • base classes and derived classes: dervied inherits base classes
  • traditionally, the derived class will create overrides
  • classes are the blueprint in which the objects are created from
  • This is truely not in javascript, but can be imitated via prototypal inheritance with the function constructor pattern

Pros/Cons

  • "simple"/"verbose"
  • objects/classes create objects
  • mutable/immutable
  • dynamic/not dynamic
  • redundancy
  • composition
